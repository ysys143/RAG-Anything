*   **WebSocket 메시지 크기:** 실시간 모니터링 시 세션 데이터가 너무 많으면 브라우저로 한 번에 보낼 수 있는 데이터량을 초과할 수 있습니다. 이 경우 `Max Binary Message Size`나 `Max Text Message Size` 값을 조정하여 처리 가능한 데이터 용량을 늘려야 합니다.

EZIS 모니터링 시스템에서 쿼리를 수행함에도 불구하고 세션이나 그래프가 실시간으로 표시되지 않는 경우, 다음과 같은 단계별 점검 및 조치가 필요합니다.

### 1. 서버 연결 상태 및 정보 확인
가장 먼저 모니터링 대상 서버와 시스템 간의 연결이 정상적인지 확인해야 합니다. 연결이 실패하면 그래프를 그릴 수 있는 데이터를 수집할 수 없습니다.
*   **조치 방법:** `Setting > Monitoring > Server` 메뉴로 이동하여 해당 서버의 체크박스를 선택한 후 **'Test Connection'** 버튼을 누릅니다. 만약 '연결 실패' 메시지가 나온다면, 서버 정보(IP, Port, 계정 등)를 다시 확인하여 수정해야 합니다 [2, 3, 4].
*   **수집 상태 확인:** 화면 우측 상단의 햄버거 버튼을 클릭하여 **'Logging Status'** 화면을 확인하십시오. 각 항목별로 'Suspend(중지)' 상태인 기능이 있다면 'Resume'을 통해 재개할 수 있습니다 [1, 2].

### 2. 세션 필터링 설정 확인
특정 조건에 따라 세션 정보가 화면에서 제외되도록 설정되어 있을 수 있습니다.
*   **비활성 세션(Inactive Session) 필터:** 설정에서 'Inactive' 세션을 제외하도록 설정한 경우, 활동 중이지 않은 세션은 표시되지 않습니다. 다만, Lock 등 경합 관계에 있는 경우에는 표시될 수 있습니다 [2].
*   **시스템 유저 필터:** `sys`, `system` 등 시스템 관리 계정의 세션 정보가 제외되도록 설정되어 있는지 확인하십시오 [2].
*   **수행 시간 필터:** 실시간 화면의 `Filter - Elapsed Time` 설정값이 너무 높게 설정되어 있으면, 해당 시간(ms) 미만으로 빠르게 수행되는 쿼리(Threads)는 화면에 표시되지 않습니다 [2, 4].

### 3. 브라우저 캐시 및 제품 패치 관련
제품이 패치되거나 기능이 추가된 경우, 이전 스크립트가 브라우저 캐시에 남아 있어 그래프가 정상적으로 표현되지 않을 수 있습니다.
*   **조치 방법:** 크롬 브라우저에서 **F12**를 눌러 개발자 모드로 진입한 후, 새로고침 아이콘을 마우스 오른쪽 버튼으로 클릭하여 **'강력 새로고침'**을 수행하십시오 [2, 3].

### 4. 라이선스 및 성능 데이터 저장 공간 점검
*   **저장 공간 부족:** 성능 데이터 저장 공간이 부족할 경우 설정된 용량을 초과하면 오래된 데이터가 자동으로 삭제될 수 있습니다. `Setting > Advanced > General`에서 저장 기간과 디스크 파티션 크기 설정을 확인하십시오 [2, 3, 4].
*   **라이선스 만료:** 라이선스가 만료된 경우 로그인은 제약되지만, 모니터링 및 성능 지표 수집은 정상적으로 동작하도록 설계되어 있습니다 [2, 3].
